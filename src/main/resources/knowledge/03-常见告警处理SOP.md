# 常见告警处理 SOP

以下为常见告警的标准处置步骤（SOP），研判相关告警时请参考。所有步骤均为人工处置建议，Agent 当前只读、不执行任何操作。

## 磁盘使用率过高（disk_usage_high）

含义：主机磁盘使用率超过阈值，可能导致写入失败、服务异常。

处置步骤：
1. 通过告警详情确认主机（来源对象/对象地址）和触发阈值。
2. 登录主机定位占用大的目录与文件（日志、临时文件、备份、core dump 等）。
3. 优先清理可安全删除的内容：过期日志、临时文件、无用归档。
4. 若为业务数据正常增长，评估扩容或挂载新盘，而非反复清理。
5. 确认使用率回落到阈值以下后，再将告警置为已解决/已关闭。

升级条件：清理后短时间内再次触发、或使用率持续逼近 100%、或影响核心服务，应升级为高级别并通知责任人。

## CPU 使用率过高

处置步骤：
1. 确认是单进程占用还是整体负载升高。
2. 排查异常进程、突发流量、定时任务叠加、死循环等。
3. 区分临时峰值与持续高负载：持续高负载需扩容或优化。

## 内存使用率过高 / OOM

处置步骤：
1. 确认是缓存占用、内存泄漏还是真实需求增长。
2. 检查近期发布或配置变更。
3. 必要时重启释放并安排根因排查，避免反复 OOM。

## 服务/进程不可用

处置步骤：
1. 确认进程是否存活、端口是否监听、健康检查是否通过。
2. 查看服务日志定位启动失败或崩溃原因。
3. 恢复服务后确认依赖（数据库、中间件）正常，再关闭告警。

## 通用研判提示

- 先看告警详情和累计次数，判断是偶发还是反复。
- 用相似告警功能判断是否为同类批量问题（如同一批主机）。
- 没有把握时如实说明需要进一步排查，不要臆造原因或结论。
