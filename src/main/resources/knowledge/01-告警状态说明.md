# 告警状态说明

优云 Alert 平台的告警状态及其处置含义如下，研判和给出建议时请据此判断告警当前处于哪个处置阶段。

| 状态 | 含义 | 建议处置方向 |
|------|------|--------------|
| 未接手 | 告警新产生，尚无人认领处理 | 提醒尽快接手；高级别告警应优先认领 |
| 已确认 | 已有人确认告警真实存在并准备处理 | 关注处理进展，必要时升级 |
| 处理中 | 责任人正在处理 | 跟踪处理时长，超时未解决应升级 |
| 已解决 | 问题已被处理，等待确认恢复 | 确认指标恢复正常后再关闭 |
| 已关闭 | 告警闭环，不再活跃 | 仅供历史追溯，默认列表不展示 |

## 研判要点

- 默认告警列表只展示**未关闭**的告警（未接手/已确认/处理中/已解决），按最近发生时间倒序。
- 同一对象反复产生告警且累计次数（count）较高，说明问题未根治，应优先排查根因而非反复关闭。
- “未接手”且为严重级别的告警停留时间过长，是重点风险，应在研判中明确提示。
- 状态只反映处置流程，不代表问题真实是否解决；务必结合指标和累计次数判断。
